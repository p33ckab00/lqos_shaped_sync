LibreQoS.py --updateonly
LibreQoS apply policy
Non-dry-run syncs automatically run LibreQoS.py --updateonly after successful file writes when changes are detected. Dry-run never applies. If an apply fails after files were written, LQoSync keeps a pending apply marker and retries on the next non-dry-run cycle. Use Force LibreQoS Apply only when you need to apply current files immediately without waiting for a file diff.
