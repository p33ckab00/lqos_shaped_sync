Default safe update

This is the recommended production update. It pulls source code from GitHub, preserves operator-owned data, migrates only missing safe config defaults, reapplies permissions, and restarts the service.

cd /opt/lqosync
sudo UPDATE_POLICY=preserve_and_migrate bash upgrade.sh
Preserved: /opt/libreqos/src/config.json, ShapedDevices.csv, network.json, /opt/lqosync/users.json, .env, state, logs, and backups.
Diverged history recovery

Use this when git pull fails because local and remote histories diverged. It backs up the local branch, resets app code to GitHub main, and keeps live operational files outside the app-code history.

cd /opt/lqosync
sudo git branch backup-before-reset-$(date +%F_%H%M%S)
sudo git fetch origin main
sudo git reset --hard origin/main
sudo LQOSYNC_INIT_POLICY=preserve_existing bash install.sh
sudo systemctl restart lqos_shaped_sync
Adopt existing ZIP/manual install

Use this if /opt/lqosync exists but was not installed by Git clone. It creates a safety backup, pulls the GitHub source, restores local users/env/logs/state, and runs preserve-existing install.

cd /opt
curl -fsSL https://raw.githubusercontent.com/p33ckab00/lqos_shaped_sync/main/install-from-github.sh -o /tmp/install-lqosync.sh
sudo EXISTING_INSTALL_ACTION=adopt bash /tmp/install-lqosync.sh
Update policy options
pull_onlyFetch/pull code only. No install, migration, dependency update, or restart.
code_onlyUpdate app code but avoid touching live config/runtime files.
preserve_and_migrateRecommended default. Preserve live files and add safe missing defaults.
refresh_with_backupMajor repair/refresh with backups, permissions, and service reinstall.
factory_resetDanger mode; requires explicit confirmation.
Why updates are not executed directly from this page

Git updates can stop services, rewrite application code, repair permissions, and migrate config. LQoSync shows exact commands here so the operator can run them over SSH with full terminal visibility, sudo control, backups, and rollback context.
